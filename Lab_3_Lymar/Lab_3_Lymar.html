def bubble_sort(array):
    n = len(array)
    for i in range(n):
        for j in range(0, n-i-1): # Оскільки останній i елемент вже на місці
            if array[j] > array[j+1] :
                array[j], array[j+1] = array[j+1], array[j]

import time

def bubble(nums):
    swapped = True
    a = 0
    while swapped:
        swapped = False
        for i in range(len(nums) - 1 - a):  # Исправлено: удалите инкремент a из цикла
            if nums[i] > nums[i + 1]:
                nums[i], nums[i + 1] = nums[i + 1], nums[i]
                swapped = True
        a += 1  # Исправлено: сдвигаем 'a' сюда

def insertionsort(nums):  # Функция переименована согласно её вызову ниже
    for j in range(1, len(nums)):
        key = nums[j]
        i = j - 1
        while i >= 0 and nums[i] > key:
            nums[i + 1] = nums[i]
            i -= 1
        nums[i + 1] = key

Відповіді на запитання:

  1. Відмінність між О-нотацією та Омега-нотацією полягає в тому, що О-нотація встановлює верхню межу, тобто наскільки погано алгоритм може поводитися, а Омега-нотація встановлює нижню межу, наскільки добре.

  2. Алгоритм сортування вставками, коли масив вже відсортований, має лінійну тимчасову складність, тобто O(n), в такому випадку не потрібні дії для перестановки елементів, проводиться лише порівняня, що призводить до кількості операцій рівними n.

  3. Ефективний алгоритм - це алгоритм, який може ефективно вирішувати проблеми шляхом використання обмежених ресурсів, таких як час виконання та пам'ять, навіть при рості розміру вхідних даних.

  4. Головний параметр задачі є важливим у визначенні часової складності алгоритмів. Із зростанням головного параметра може зростати час виконання або використання пам'яті алгоритму. Цей параметр зазвичай визначається числом або розміром вхідних даних або складністю самої задачі.

  5. $O(n^3)$
